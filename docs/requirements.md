# Vaqlo — требования

Локальный «чёрный ящик» для macOS: записывает микрофон и системный звук (всё, что играет в динамиках/наушниках), складывает на диск, транскрибирует локально и превращает в markdown-заметки с метаданными — чтобы потом искать, саммаризировать и ничего не забывать. Аналог Plaud/Granola, но на лаптопе, бесплатно и без облака.

## Принципы

- **Локально и бесплатно.** Никаких облачных API, подписок, отправки звука наружу.
- **Две дорожки раздельно.** Микрофон и системный звук пишутся в отдельные файлы — это и качество транскрипции, и грубая диаризация («я» vs «собеседник») бесплатно.
- **Слабая связанность.** Рекордер, транскрайбер и генератор заметок — независимые компоненты, общаются через файловую систему.
- **Приватность под контролем.** Запись включается только явно (хоткей/кнопка), всегда видно, что идёт запись (иконка в menu bar).

## Фазы

### Фаза 1 — Рекордер (текущая)

Menu bar приложение на Swift (`recorder/`).

Функциональные требования:

- **Запуск/остановка записи**:
  - глобальная горячая клавиша **⌥⌘R** (работает из любого приложения);
  - кнопка в menu bar (иконка) с выпадающей панелью контролов: статус, Start/Stop, открыть папку записей, автозапуск при логине, Quit.
- Иконка отражает состояние: обычная — простой, красная/filled — идёт запись.
- **Захват звука**:
  - микрофон — через `AVAudioEngine`;
  - системный звук — через **Core Audio process taps** (macOS 14.4+), системный микс целиком; без BlackHole и прочих драйверов.
- **Файлы**: AAC (`.m4a`), чанки по 5 минут, чтобы падение/перезагрузка не теряли больше 5 минут и чтобы транскрайбер мог работать инкрементально.
- **Раскладка на диске**: `~/VaqloRecordings/<YYYY-MM-DD>/<HHMMSS>/`
  - `mic_0001.m4a`, `mic_0002.m4a`, …
  - `sys_0001.m4a`, `sys_0002.m4a`, …
  - `session.json` — метаданные сессии.
- **Метаданные** (`session.json`): время старта/остановки, список чанков с таймстампами, таймлайн активного приложения (сэмплируется раз в 5 с: bundle id + имя) — чтобы транскрипт потом знал, что «в 14:02 был активен Zoom».
- Разрешения: микрофон (`NSMicrophoneUsageDescription`) + System Audio Recording (`NSAudioCaptureUsageDescription`). Screen Recording не нужен.

Нефункциональные:

- Запись дешёвая по CPU/батарее; никакой обработки звука в реальном времени.
- Приложение без Dock-иконки (`LSUIElement`), только menu bar.
- Сборка без Xcode-проекта: SwiftPM + скрипт, собирающий `.app` bundle (ad-hoc подпись).

### Фаза 2 — Полноценное приложение (текущая)

Рекордер превращается в обычное оконное приложение (menu bar остаётся), всё управление — внутри него, без внешних скриптов.

- **Хранение внутри приложения**: записи переезжают в `~/Library/Application Support/Vaqlo/recordings/…`; существующие записи из `~/VaqloRecordings` мигрируются автоматически.
- **Окно с таймлайном**: суточный и недельный вид; сессии — блоки на шкале времени; клик по сессии — детали (метаданные, статус, транскрипт).
- **Транскрибация в приложении**:
  - движок — `whisper.cpp` (whisper-cli, Metal), бинарь собирается при сборке и кладётся внутрь `.app`;
  - модель Whisper скачивается прямо из приложения (настройки → Модели, с прогрессом); пресеты: large-v3-turbo (рекомендуется), small, base;
  - запуск вручную (кнопка у сессии / «транскрибировать всё») или по расписанию из настроек (выкл / каждые N часов / ежедневно в указанное время);
  - язык — автоопределение (русский + английский).
- **Жизненный цикл аудио**: после успешной транскрибации `.m4a` уезжают в Корзину; приложение помнит, что положило туда, и само удаляет эти файлы через 7 дней (настраивается).
- **Транскрипты**: хранятся рядом с метаданными сессии (`transcript.md` + `transcript.json`), просматриваются прямо в приложении; кнопка «Экспорт» выгружает md в локальную папку, выбранную в настройках.
- **Control Center**: контрол-тоггл записи в панели контролов macOS 26 (WidgetKit ControlWidget); показывает статус («Запись с 14:02») и число сессий, ждущих транскрибации.
- Объединение дорожек: сегменты mic («Я») и системной дорожки (подпись = активное приложение в этот момент) сливаются в один хронологический транскрипт.

UX-требования (итерация 2):

- Левый клик по иконке в menu bar — открыть окно приложения; правый клик — меню.
- Иконка в menu bar — эмодзи: отдельный для простоя (по умолчанию 😶) и для записи (по умолчанию 🙂 — запись не бросается в глаза); выбираются в настройках.
- Горячая клавиша настраивается в настройках (рекордер сочетания).
- Деталка сессии: начало/конец, длительность, список приложений, классификация (видеовстреча Zoom/Teams/Meet… / звук с компьютера / офлайн-разговор у компьютера), удаление в Корзину с подтверждением.
- Корзина внутри приложения: список элементов, таймер «само удалится через N дн.», чтение транскриптов удалённых сессий, восстановление, удаление сейчас, очистка всей корзины.
- Все кнопки дают мгновенный фидбек: спиннер/прогресс сразу после нажатия, повторное нажатие невозможно.
- Аудио удаляется автоматически через N дней (чистка ежечасно); транскрипты не удаляются никогда, только вручную.

### Фаза 3 — Обогащение

Сделано:

- ✅ VAD (silero) перед транскрибацией — тишина отбрасывается, галлюцинации Whisper («Thank you» на пустых местах) исчезают; модель скачивается автоматически.
- ✅ Диаризация обеих дорожек (FluidAudio, CoreML/ANE): реплики разделяются по голосам.
- ✅ Идентификация голосов: библиотека голосов (`voices.json`) — спикера переименовываешь один раз («Спикер 1» → «Иван»), отпечаток запоминается, дальше распознаётся автоматически (порог 0.65; 0.5–0.65 → «Иван?»). Доминирующий голос микрофонной дорожки — «я», отпечаток копится автоматически. Отпечатки сессии лежат в `speakers.json` рядом с транскриптом.

Осталось:

- Имена: подтягивать участников из событий календаря, пересекающихся по времени с записью.
- Список процессов, реально игравших звук (Core Audio: per-process activity) — точнее, чем «активное приложение».
- Управление библиотекой голосов в настройках (посмотреть/удалить голос).

### Фаза 4 — Поиск и саммаризация

Сделано:

- ✅ Саммаризация локальной LLM: встроенный `llama-completion` (llama.cpp, Metal) + скачиваемые в настройках GGUF-модели (Qwen3 4B — рекомендуется, Llama 3.2 3B/1B). Кнопка «Сделать саммари» у сессии или авто-режим после транскрибации (настройка). Формат: TL;DR / Решения / Действия; длинные транскрипты — по кускам с финальной сводкой. Результат в `summary.md`, включается в экспорт.
- ✅ Модели диаризации (FluidAudio) видны в настройках: статус, размер, удаление.
- ✅ Навигация стрелками ↑/↓ между записями дня (в недельном виде — по всей неделе).

- ✅ Автодетект встреч: Core Audio process-objects API (`kAudioProcessPropertyIsRunningInput`) замечает, что приложение заняло микрофон. Режимы в настройках: выкл / уведомить с кнопкой «Записать» / записывать автоматически. В авто-режиме — опциональная остановка через 20 с после освобождения микрофона. Системные уведомления через UNUserNotificationCenter.

- ✅ Поиск по всем транскриптам: поле в шапке окна, полнотекстовый поиск по `transcript.json` всех сессий, результаты со сниппетами и числом совпадений; клик открывает сессию.
- ✅ Аудиоплеер: микрофонная и системная дорожки сшиваются в один таймлайн (AVMutableComposition) и звучат вместе; транспортная панель (play/pause, скраббер, время) в деталке; клик по времени реплики перематывает на её момент. Работает, пока аудио существует (до транскрибации и в окне хранения Корзины); иначе — «Аудио удалено, остался транскрипт».

- ✅ Плеер v2: перемотка ±15 с, скорость (0.75–2×, с коррекцией высоты тона), раздельный мьют дорожек (микрофон / компьютер через AVMutableAudioMix), подсветка и автоскролл к реплике, играющей сейчас.
- ✅ Статистика и хранилище (вкладка настроек): число сессий по статусам, записано часов всего и за неделю, разбивка занятого места (аудио / Корзина / транскрипты / модели Whisper-LLM / модели диаризации), кнопка «Открыть папку данных».
- ✅ Онбординг (первый запуск + меню «Разрешения и помощь»): пошаговый экран с микрофоном, системным звуком, уведомлениями, автозапуском — со статусами и кнопками.
- ✅ Профили голосов (вкладка «Голоса»): список запомненных голосов с числом образцов и пометкой «это вы», переименование (со слиянием отпечатков при совпадении имён) и удаление.

## Открытые вопросы

- Настраиваемость хоткея (v1 — захардкожен ⌥⌘R).
- Пауза (отдельно от стопа) — нужна ли.
- Куда класть md-заметки (Obsidian vault?) — уточнить к фазе 3.
- Юридическое: при записи звонков в ряде юрисдикций требуется согласие собеседников — ответственность на пользователе, приложение всегда показывает индикатор записи.
